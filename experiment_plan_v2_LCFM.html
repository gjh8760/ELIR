Retinex-Guided Latent Consistency Flow Matching for Low-Light Image Enhancement

NeurIPS 2026 실험 계획서 (v2 — ELIR 기반)

변경 사항: MeanFlow(JVP 연산 과다) → ELIR의 LCFM(Latent Consistency Flow Matching) 방식으로 전환
핵심 방법: Latent space에서 OT-CFM + consistency distillation, Retinex 분해 기반 dual-flow
총 소요: ~10주 | 목표: NeurIPS 2026

성능 타깃 (LOL-v1 PSNR 기준)

구분 PSNR
현재 Phase 1 (Direct LCFM baseline) 22.10 dB
SOTA 근방 (LLIE 최신 기법 수준, 논문 경쟁력 마지노선) 25.5 ~ 26+ dB
필요 개선폭 +3 dB 이상

Phase 2 이후 모든 단계의 설계 결정은 이 +3dB 갭을 좁히는 것을 1순위로 삼아야 합니다. 단순히 "Phase 1 대비 0.5dB 개선"만으로는 SOTA 경쟁이 불가능하므로, 개선폭이 기대 이하이면 조기에 pivot (예: pixel space, VAE 교체, 더 큰 Eω/flow, data augmentation/추가 데이터) 해야 합니다.

방법론 요약: ELIR이 하는 것과 우리가 다르게 할 것

ELIR의 핵심 파이프라인

  1. Tiny AutoEncoder로 이미지를 latent space로 압축 (512×512 → 64×64×16)
  2. Latent MMSE estimator: coarse L2 predictor로 latent 수준에서 1차 복원
  3. LCFM: OT-conditional flow matching + consistency flow matching을 결합하여 few-step(1-4 NFE) 생성
  4. 전부 CNN 기반, transformer 없음

우리가 다르게 할 것 (novelty)

  • ELIR: 단일 flow로 LQ→HQ 전체를 한 번에 처리
  • Ours: Retinex 분해 후 I-flow, R-flow를 각각 학습 + cross-guidance로 coupling
  • ELIR: generic restoration (face, SR, denoising 등)
  • Ours: LLIE에 특화된 physics-informed 설계
  • 추가: perception-distortion tradeoff를 σ로 제어 (PMRF 아이디어 차용)

Phase 0: LCFM Feasibility on LLIE (1주)

목적

ELIR 방식의 LCFM이 low-light enhancement에서 작동하는지 최소 비용으로 검증. MeanFlow와 달리 JVP가 없으므로 구현 리스크는 낮지만, LLIE 특화 세팅에서의 성능을 확인해야 함.

실험 설계

  • 데이터: LOL-v1 (485 pairs), 256×256 random crop
  • 구조: ELIR 논문의 파이프라인을 최대한 그대로 재현하되 LLIE용으로 세팅
  • AutoEncoder: Tiny AutoEncoder (TAESD, SD3 VAE의 경량 버전, 1.2M params)
    • 설치: pip install diffusers → AutoencoderTiny.from_pretrained("madebyollin/taesd3")
    • 입력 이미지를 latent space로 인코딩: 3×H×W → 16×(H/8)×(W/8)
  • Coarse estimator: 소형 CNN (3-layer conv, latent→latent L2 regression)
  • Flow network: 소형 U-Net (latent space에서 동작, ~5M params)

Training 과정 (ELIR 방식)

Step 1: AutoEncoder freeze, coarse estimator gφ 학습
  z_lq = E(x_lowlight)         # encode low-light to latent
  z_hq = E(x_normal)           # encode normal to latent (GT)
  z_coarse = gφ(Eω(x_lowlight)) # coarse estimation
  loss_coarse = MSE(z_coarse, z_hq)

Step 2: Flow matching 학습 (coarse 결과를 source로)
  z_0 = z_coarse + σ_s · ε     # coarse estimate + small noise
  z_1 = z_hq                    # target
  z_t = t·z_1 + (1-(1-σ_min)·t)·z_0  # OT interpolation
  loss_fm = MSE(vθ(z_t, t), z_1 - (1-σ_min)·z_0)

Step 3: Consistency distillation (optional, few-step 개선)
  teacher의 multi-step ODE 결과를 student가 fewer step으로 근사

Inference

z_lq = E(x_lowlight)
z_coarse = gφ(Eω(x_lowlight))
z_0 = z_coarse + σ_s * noise
# Euler solve: 1~4 steps
for step in range(num_steps):
    z = z + dt * vθ(z, t)
x_enhanced = D(z)  # decode back to pixel space

Go/No-Go

  • ✅ 4-NFE에서 LOL-v1 PSNR ≥ 19dB → proceed
  • ✅ 시각적으로 밝기 개선 + noise 감소 확인 → proceed
  • ❌ latent space 복원이 pixel space 대비 심하게 열화 → AutoEncoder 교체 검토
  • ❌ 학습 불안정 → σ_s, σ_min hyperparameter 조정

Phase 1: Direct LCFM LLIE Baseline (2주)

목적

Retinex 없이 direct LCFM으로 LLIE baseline 성능 확립. Phase 0의 toy 실험을 full-scale로 확장.

Network Architecture

AutoEncoder (teacher, frozen)

  • Tiny AutoEncoder TAESD3 (madebyollin/taesd3): encoder + decoder
  • 압축비: 8× spatial, 16ch latent
  • 256×256 입력 → 32×32×16 latent
  • teacher_cfg.name: taesd, trainable: False (decoder도 inference에서만 사용)

Trainable Encoder Eω (tiny_enc)

  • TAESD3 encoder 가중치로 초기화 후 학습 가능(enc_cfg.trainable: True)
  • LQ 이미지 전용으로 fine-tuning (degradation에 적응)

Coarse Estimator gφ (mmse, ~5.5M)

  • RRDBNet (mmse_cfg.name: rrdbnet): 3개의 RRDB 블록(각 RRDB = 3× ResidualDenseBlock_5C), c_inout=16, c_hid=96, n_rrdb=3
  • Input/Output: 16ch latent → 16ch latent (residual 연결)
  • L2 loss로 학습(실제로는 아래 통합 loss의 첫 항)

Flow Network vθ (lunet, ~29.6M)

  • Latent space U-Net (fmir_cfg.name: lunet)
  • Input: z_t만(16ch) — z_coarse는 flow의 source z_0로 사용되며 채널 concat은 아님
  • Output: 16ch (velocity)
  • ch_mult=[1,2,1,2], hid_channels=128 (base 128), n_mid_blocks=3, skip connections 있음
  • Time conditioning: sinusoidal positional embedding(t_emb_dim=160) → MLP → ResnetBlock 출력에 per-channel bias로 더함 (AdaGN 아님)

LCFM Training Loss (실제 구현: l2_cfm_mse_loss)

한 번의 loss 안에 MMSE + segment-endpoint consistency + pixel-space MSE가 α, β 계수로 결합되어 있습니다(2-phase 분리 학습 아님).

# 1) MMSE (coarse) loss (latent space)
X_hq    = E_teacher(x_hq).detach()
X_lq    = Eω(x_lq)
X_mmse  = gφ(X_lq)
loss    = MSE(X_hq, X_mmse)

# 2) Flow source 구성: coarse + small noise
X0 = X_mmse.detach() + σ_s * ε,   ε ~ N(0, I)

# 3) t 샘플링 & segment endpoint
# [0,1]을 K=k_steps=5 구간으로 분할, t가 속한 segment의 오른쪽 끝 seg_end
segments   = linspace(0, 1, K+1)
seg_end    = segments[searchsorted(segments, t, side="left").clamp(min=1)]
X_ends     = (1 - (1-σ_min)*seg_end) * X0 + seg_end * X_hq

# 4) OT-CFM 경로 위의 두 점 Xt, X_{t+dt}에서 velocity 예측
Xt = (1 - (1-σ_min)*t) * X0 + t * X_hq
v0 = vθ(Xt,  pos_emb(t))
with no_grad():
    Xr = (1 - (1-σ_min)*(t+dt)) * X0 + (t+dt) * X_hq
    v0_ = vθ(Xr, pos_emb(t+dt))

# 5) Segment-endpoint consistency: 두 점에서 seg_end까지 extrapolate한 결과가 일치하도록
f0  = Xt + (seg_end - t)     * v0
f0_ = (r_less) * (Xr + (seg_end - (t+dt)) * v0_) + (~r_less) * X_ends    # r_less = (t+dt) < seg_end
loss += (1-β) * ( MSE(f0, f0_) + α * MSE(v0, v0_) )

# 6) Pixel-space MSE (decoder 경유)
X1       = f0.detach() + (1 - seg_end) * vθ(f0.detach(), pos_emb(seg_end))
x_hq_hat = D_teacher(X1)
loss += β * MSE(x_hq, x_hq_hat)

참고: 수학적으로 target velocity = X_hq − (1−σ_min)·X0와 동치이지만, 코드에서는 velocity-MSE를 직접 쓰지 않고 segment 끝점 일치(consistency) 형태로 표현되어 있습니다.

Training Details

  • Optimizer: AdamW, lr=2e-4
  • Batch size: 8 (LOL-v1 our485, 256×256 random crop)
  • Schedule: 1000 epochs 단일 스케줄 (≈ 485/8 × 1000 ≈ 60K steps), EMA decay 0.999
  • Validation: 20 epoch마다 (LOL-v1 eval15)
  • Hyperparameters (config 기준): σ_min = 1e-5, σ_s = 0.025, dt = 0.05, k_steps = 5, α = 0.001, β = 0.001, t_emb_dim = 160

Evaluation

  • NFE sweep: 1, 2, 4, 8
  • Metrics: PSNR, SSIM, LPIPS, NIQE
  • Baselines: RetinexNet, Retinexformer, LLFlow, DiffLL

Go/No-Go

  • ✅ 2-NFE에서 PSNR ≥ 20dB on LOL-v1 → Phase 2
  • ✅ 4-NFE에서 Retinexformer에 근접 (차이 < 1.5dB) → 유망
  • ❌ 성능 부족 → flow network 크기 확대 또는 pixel space 전환 검토

Phase 2: Retinex Decomposition + Component Flows (2주)

목적

Retinex 분해 후 Illumination과 Reflectance에 각각 별도의 LCFM을 적용하여 Phase 1 대비 성능 향상을 검증.

Step 2-A: Retinex Decomposer 선택 및 구현

3가지 옵션을 순서대로 시도:

Option 1: RetinexNet Decom-Net (추천, 먼저 시도)

RetinexNet의 Decom-Net은 가장 널리 검증된 학습 기반 decomposer입니다.

구조: - Input: RGB image (3ch) - 3×3 conv → feature extraction - 여러 3×3 conv + ReLU layers - 마지막 3×3 conv → sigmoid로 R(3ch)과 I(1ch) 출력 - 파라미터: ~0.04M (매우 경량)

학습 방식 (pretrained weights 사용 시 불필요): - Paired low/normal 이미지를 동시에 decompose - Loss = Reconstruction(S = R⊙I) + Reflectance consistency(R_low ≈ R_normal) + Illumination smoothness - Ground truth 없이 self-supervised로 학습

실제 사용:

# Pretrained weights 다운로드
# https://github.com/weichen582/RetinexNet (official TF)
# https://github.com/aasharma90/RetinexNet_PyTorch (PyTorch 포팅)

# 또는 KinD의 decomposer도 사용 가능 (더 정교)
# https://github.com/zhangyhuaee/KinD

decomposer = DecomNet(pretrained="retinexnet_decom.pth")
decomposer.eval()  # frozen

R_low, I_low = decomposer(x_lowlight)   # R: 3ch, I: 1ch
R_norm, I_norm = decomposer(x_normal)   # paired GT도 decompose
# x = R ⊙ I (element-wise, I는 broadcast)

Option 2: Max-Channel 기반 (가장 빠른 baseline)

학습 불필요. Retinex theory의 가장 단순한 구현:

def simple_decompose(img):
    # img: [B, 3, H, W], range [0, 1]
    I = img.max(dim=1, keepdim=True)[0]  # [B, 1, H, W]
    I_smooth = gaussian_blur(I, kernel_size=15, sigma=3.0)
    I_smooth = I_smooth.clamp(min=1e-4)  # avoid division by zero
    R = img / I_smooth  # [B, 3, H, W]
    R = R.clamp(0, 1)
    return R, I_smooth

장점: 즉시 사용, 외부 의존 없음 단점: smooth kernel size가 고정이라 edge 근처에서 artifact, noise가 R에 그대로 남음

Option 3: KinD Decomposer (가장 정교)

KinD의 decomposer는 RetinexNet보다 정교한 구조 (multi-scale, attention). 성능이 가장 좋지만 dependency가 복잡. Phase 2 후반에 Option 1 대비 성능 차이를 ablation으로 확인.

추천 순서

  1. Option 2 (max-channel)로 하루 만에 빠르게 검증
  2. 효과 있으면 Option 1 (RetinexNet Decom-Net)으로 교체하여 본 실험
  3. Ablation에서 Option 3 (KinD)까지 비교

Step 2-B: Component-wise LCFM in Latent Space

Retinex 분해 결과를 각각 latent space로 보내서 별도 flow를 학습:

# Decompose in pixel space
R_low, I_low = decomposer(x_lowlight)   # R: 3ch, I: 1ch (→ 3ch로 repeat)
R_norm, I_norm = decomposer(x_normal)

# I를 3ch로 확장 (AutoEncoder가 3ch 입력을 기대)
I_low_3ch = I_low.repeat(1, 3, 1, 1)
I_norm_3ch = I_norm.repeat(1, 3, 1, 1)

# Encode to latent
z_R_low = E(R_low)       # reflectance latent
z_R_norm = E(R_norm)
z_I_low = E(I_low_3ch)   # illumination latent
z_I_norm = E(I_norm_3ch)

# Coarse estimates in latent
z_R_coarse = gφ_R(Eω_R(R_low))
z_I_coarse = gφ_I(Eω_I(I_low_3ch))

주의: I가 1ch인 문제 AutoEncoder(TAESD)는 3ch RGB를 기대합니다. 해결법 3가지: - (A) I를 3ch로 repeat → 가장 단순, 다소 낭비 - (B) I 전용 소형 1ch→latent encoder를 별도 학습 → 깔끔하지만 공수 증가 - (C) I는 latent space 사용 안 하고 pixel space에서 직접 처리 → I가 smooth하므로 latent 불필요할 수 있음

추천: (C) — I는 pixel space, R만 latent space

근거: Illumination은 low-frequency하고 smooth해서 latent compression의 이점이 적고, 1ch이라 AutoEncoder와 호환도 어색합니다. 반면 Reflectance는 3ch RGB이고 texture/noise가 복잡하여 latent space flow가 효과적.

I-flow: pixel space에서 직접 LCFM (경량 CNN, ~3M)
  Source: I_low → Target: I_norm
  매우 smooth한 변환 → 1-2 NFE로 충분

R-flow: latent space에서 LCFM (U-Net, ~10M)
  Source: z_R_coarse → Target: z_R_norm
  복잡한 noise/texture 복원 → 2-4 NFE 필요

Step 2-C: Reconstruction

# Inference
I_restored = I_low + vel_I  # pixel space, 1-NFE
z_R_restored = euler_solve(vθ_R, z_R_coarse, steps=4)  # latent, 4-NFE
R_restored = D(z_R_restored)  # decode to pixel

x_enhanced = R_restored * I_restored  # Retinex reconstruction

Phase 3: Coupled Dual-Flow with Cross-Guidance (2주)

목적

Phase 2의 독립적 I-flow, R-flow를 coupling하여 cross-component interaction 효과 검증. 논문의 핵심 novelty.

Cross-Guidance 설계

Phase 2에서 I는 pixel space, R은 latent space로 결정했으므로, 두 flow의 정보 교환 방식을 설계해야 합니다.

Architecture: Asymmetric Coupled Flow

class CoupledRetinexFlow(nn.Module):
    def __init__(self):
        # I-flow: pixel space, lightweight
        self.I_net = LightweightUNet(in_ch=2, out_ch=1, base_ch=32)
        # in_ch=2: I_t(1ch) + I_guidance_from_R(1ch)

        # R-flow: latent space, heavier
        self.R_net = LatentUNet(in_ch=32+16, out_ch=16, base_ch=64)
        # in_ch=32: z_R_t(16ch) + z_R_coarse(16ch)
        # +16은 아래 R_guidance_encoder의 출력

        # Cross-guidance modules
        self.R_to_I_guidance = nn.Sequential(
            nn.Conv2d(16, 32, 3, 1, 1), nn.SiLU(),
            nn.AdaptiveAvgPool2d(output_size=None),  # spatial align
            nn.Conv2d(32, 1, 1, 1, 0), nn.Sigmoid()
        )  # latent R feature → pixel space I guidance map

        self.I_to_R_guidance = nn.Sequential(
            nn.Conv2d(1, 16, 3, 2, 1), nn.SiLU(),
            nn.Conv2d(16, 16, 3, 2, 1), nn.SiLU(),
            nn.Conv2d(16, 16, 3, 2, 1), nn.SiLU(),
        )  # pixel I → latent-size feature (downscale 8x to match latent)

    def forward(self, I_t, z_R_t, t, I_low, z_R_coarse):
        # Step 1: R → I guidance
        # "reflectance의 edge/texture 정보로 illumination 조절을 guide"
        R_guidance = self.R_to_I_guidance(z_R_t)
        # upsample to pixel resolution
        R_guidance_pixel = F.interpolate(R_guidance, size=I_t.shape[-2:])
        I_input = torch.cat([I_t, R_guidance_pixel], dim=1)
        vel_I = self.I_net(I_input, t)

        # Step 2: I → R guidance
        # "현재 illumination 상태로 noise 제거 강도를 adaptive하게 조절"
        I_guidance = self.I_to_R_guidance(I_t)
        R_input = torch.cat([z_R_t, z_R_coarse, I_guidance], dim=1)
        vel_R = self.R_net(R_input, t)

        return vel_I, vel_R

Cross-guidance의 직관: - R→I guidance: reflectance의 edge 정보를 illumination flow에 전달 → edge-aware illumination adjustment (edge 경계에서 halo artifact 방지) - I→R guidance: 현재 시점의 illumination 밝기를 reflectance flow에 전달 → 어두운 영역(I가 낮은 곳)에서 denoising을 더 강하게, 밝은 영역에서는 약하게

Training

# Decompose
R_low, I_low = decomposer(x_lowlight)
R_norm, I_norm = decomposer(x_normal)

# Encode R to latent
z_R_low = E(R_low)
z_R_norm = E(R_norm)
z_R_coarse = gφ_R(Eω_R(R_low)).detach()  # Phase 2에서 학습한 것 재사용

# OT interpolation (same t for both flows)
t = torch.rand(B, 1, 1, 1)
I_t = (1-t) * I_low + t * I_norm
z_R_t = (1-t) * (z_R_coarse + σ_s * noise) + t * z_R_norm

# Forward
vel_I_pred, vel_R_pred = coupled_model(I_t, z_R_t, t, I_low, z_R_coarse)

# Targets
target_I = I_norm - I_low
target_R = z_R_norm - (1-σ_min) * (z_R_coarse + σ_s * noise)

# Loss
loss_I = MSE(vel_I_pred, target_I)
loss_R = MSE(vel_R_pred, target_R)
loss = loss_I + λ * loss_R  # λ=1.5 (R이 더 어려우므로)

설정 R→I I→R 비교 목적
A: Independent (Phase 2) ✗ ✗ baseline

C: I→R only ✗ ✓ illumination-adaptive denoising

Go/No-Go

  • ✅ D > A (PSNR ≥ 0.3dB 또는 LPIPS 유의미한 개선) → coupling 효과 검증
  • ❌ coupling 효과 미미 → Phase 2 결과(independent)로 논문 작성

Phase 4: PMRF-style Controllable Enhancement (1.5주)

목적

단일 모델로 perception-distortion tradeoff를 σ 파라미터로 제어.

설계

Phase 3의 coupled model을 2-stage로 확장:

Stage 1: Latent MMSE Estimator (이미 Phase 2에서 학습 완료)

  • coarse estimator gφ가 E[z_R_norm | R_low]에 수렴
  • L2 loss → PSNR 최대화, but smooth/blurry

Stage 2: Coupled Flow에서 σ 조절

# Inference time에 σ_s를 조절
# σ_s 작으면: source ≈ coarse estimate → 변화 적음 → high PSNR, smooth
# σ_s 크면: source에 noise 많이 추가 → flow가 더 많이 생성 → sharp but less faithful

for sigma in [0.0, 0.05, 0.1, 0.2, 0.3, 0.5]:
    z_R_0 = z_R_coarse + sigma * torch.randn_like(z_R_coarse)
    z_R_restored = euler_solve(coupled_model.R_flow, z_R_0, steps=4)
    # evaluate PSNR, SSIM, LPIPS, FID

주의사항

  • Tiny AutoEncoder(TAESD)의 reconstruction 품질을 먼저 확인하세요. LOL 이미지에서 E(D(x)) ≈ x인지 체크. 만약 reconstruction error가 크면 이것이 성능의 bottleneck이 됩니다.
  • Retinex decomposer는 반드시 freeze하세요 (joint training 시 decomposition이 degenerate할 수 있음). Ablation에서 freeze vs. joint-finetune 비교는 가능.
  • LOL-v1이 485장으로 작으므로 overfitting 주의. Latent space에서 학습하면 effective image 크기가 줄어 overfitting 위험이 더 높을 수 있습니다. Data augmentation 적극 활용.
  • IllumFlow와의 차별점을 논문에서 명확히 서술: IllumFlow = Rectified Flow on I only + separate denoiser for R. Ours = LCFM on both I and R + coupled + latent + controllable.
