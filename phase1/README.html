Phase 1 — Direct LCFM LLIE Baseline

experiment_plan_v2_LCFM.md의 Phase 1에 해당하는 실험 결과입니다. ELIR 파이프라인(TAESD3 + RRDBNet coarse + LUNet flow)을 LOL-v1에 그대로 적용한 단일-flow 베이스라인입니다.

실험 세팅

항목 값
Task Low-Light Image Enhancement
Dataset (train) LOL-v1, our485, 485 pairs, 256×256 random crop
Dataset (eval) LOL-v1, eval15, 15 pairs
AutoEncoder (teacher) TAESD3 (madebyollin/taesd3), frozen
Trainable Encoder Eω tiny_enc (TAESD3 encoder 초기화, fine-tune)
Coarse Estimator gφ RRDBNet (c_inout=16, c_hid=96, n_rrdb=3)
Flow Network vθ LUNet (ch_mult=[1,2,1,2], hid_channels=128, n_mid_blocks=3, t_emb_dim=160)
Loss l2_cfm_mse_loss (MMSE + segment-endpoint consistency + pixel MSE, α=β=0.001)
FM hyperparams σ_min=1e-5, σ_s=0.025, dt=0.05, k_steps=5
Optimizer AdamW, lr=2e-4, EMA decay 0.999
Batch size 8
Epochs 1000 (≈ 60K steps)
Chop (eval) chop_size=256, chop_stride=224 (LOL-v1 600×400에서 겹침 inference)
Seed 2025

전체 config는 config.yaml 참고입니다.

NFE (k_steps) Sweep

학습은 k_steps=5로 했지만 추론 시에는 다른 값을 써도 동작합니다 (LCFM의 consistency loss 덕). LOL-v1 eval15에서 측정한 결과입니다.

k_steps PSNR SSIM LPIPS
1 22.0646 0.8595 0.2485
2 22.0903 0.8607 0.2451
4 22.0942 0.8608 0.2450
5 22.0998 0.8609 0.2452
8 22.0941 0.8608 0.2452
  • K=1로 내려도 PSNR 22.06 — 5-NFE 대비 0.035dB 열화만으로 few-step inference 가능.
  • K=8로 올려도 이점 없음 (segment 내부 중복 연산).
  • 상세 결과: nfe_sweep/summary.json, 각 k*/ 하위 폴더.
  • 재현: bash scripts/run_nfe_sweep.sh (GPU 2번 사용, elir 가상환경).

폴더 구성

phase1/
├── README.md              # (이 파일)
├── config.yaml            # 학습/평가 당시 config snapshot (runs/elir_llie_lolv1/conf.yml 사본)
├── metrics/
│   ├── eval_metrics.json  # 마지막 eval 메트릭 (기계가 읽기 좋은 형태)
│   └── eval_metrics.txt   # eval 메트릭 누적 로그
├── predictions/           # LOL-v1 eval15 전체 15장의 enhancement 결과 (infer.py 출력, chop_stride=128)
├── val_samples/           # 학습 중 검증 시 저장된 샘플 이미지 (epoch_XXXX/)
└── checkpoints/           # runs/elir_llie_lolv1/ 내 .pth / .ckpt 에 대한 symlink
    ├── elir.pth                         # 추론용 경량 weights
    ├── epoch=999-step=60000.ckpt        # 최종 Lightning checkpoint
    └── epoch=1-step=120.ckpt            # 초반 sanity checkpoint

재현 방법

# 평가
python eval.py --yaml configs/elir_infer_llie.yaml

# 이미지 추론 (PNG 저장)
python infer.py --yaml configs/elir_infer_llie.yaml

configs/elir_infer_llie.yaml의 arch_cfg.path가 checkpoints/elir.pth(또는 원본 runs/elir_llie_lolv1/elir.pth)를 가리키도록 해야 합니다.
