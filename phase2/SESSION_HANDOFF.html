Phase 1 → Phase 2 인수인계 문서

Phase 2 작업을 새 세션에서 이어가기 위한 컨텍스트 정리입니다. 이 파일 하나만 읽으면 지금까지의 결정/변경/결과를 모두 파악할 수 있도록 작성했습니다.

⚠️ SOTA 타깃 공지
현재 Phase 1 결과는 LOL-v1 PSNR 22.10 dB입니다. 그러나 LLIE 분야에서 논문 경쟁력을 갖추려면 PSNR 25.5 ~ 26+ dB (SOTA 근방)가 필요합니다. 즉 +3dB 이상의 개선이 반드시 Phase 2~4 안에서 나와야 합니다.

Phase 2 이후 모든 설계 결정(decomposer 선택, I/R 공간 배치, 네트워크 크기, data aug 등)은 이 +3dB 갭을 좁히는 것을 1순위로 삼아야 합니다. "Phase 1 대비 0.5dB 개선"은 SOTA 경쟁에 턱없이 부족하므로, Phase 2 결과가 24dB 미만에 머무르면 Retinex 설계 고수보다는 근본적 pivot(더 큰 VAE/pixel-space 병행/용량 증가/외부 데이터)을 우선 검토해야 합니다.

1. 전체 계획 (요약)

  • 상위 계획서: experiment_plan_v2_LCFM.html
  • 아이디어: ELIR의 LCFM(Latent Consistency Flow Matching) + Retinex 분해(I-flow / R-flow) 기반 dual-flow → LLIE 특화 + σ 기반 perception-distortion 제어 (PMRF 아이디어 차용). NeurIPS 2026 목표.
  • Phase 구성:
  • Phase 0: LCFM이 LLIE에서 동작하는지 최소 실험
  • Phase 1 (현재 완료): Retinex 없이 direct LCFM baseline 확립
  • Phase 2 (다음): Retinex 분해 후 I-flow, R-flow 각각 학습
  • Phase 3: Coupled dual-flow with cross-guidance (논문 핵심 novelty)
  • Phase 4: PMRF-style σ controllability
  • Phase 5: Full benchmark + ablation
  • Phase 6: Paper writing

2. Phase 1: 무엇을 했나

2.1 코드베이스

ELIR 원본 파이프라인을 LOL-v1에 맞춰 실행한 것이 현재 레포 상태입니다.

  • AutoEncoder (teacher, frozen): TAESD3 (madebyollin/taesd3)
  • Trainable Encoder Eω: tiny_enc (TAESD3 encoder 초기화 후 fine-tune)
  • Coarse Estimator gφ: RRDBNet (c_inout=16, c_hid=96, n_rrdb=3, ~5.5M params)
  • ELIR/models/rrdbnet.py
  • Flow Network vθ: LUNet (ch_mult=[1,2,1,2], hid_channels=128, n_mid_blocks=3, t_emb_dim=160, ~29.6M params)
  • ELIR/models/lunet.py
  • 입력은 z_t(16ch)만, z_coarse는 flow의 source z_0로 사용되며 채널 concat은 아님
  • Time conditioning: sinusoidal pos_emb → MLP → ResnetBlock 출력에 per-channel bias로 더함 (AdaGN 아님)
  • Loss: l2_cfm_mse_loss — MMSE + segment-endpoint consistency + pixel-space MSE, α=β=0.001
  • ELIR/training/losses.py L234-281
  • 수학적으로는 OT-CFM target X_hq − (1−σ_min)·X_0과 동치이지만 코드에선 segment 끝점 일치 형태로 표현

2.2 Config (configs/elir_train_llie.yaml)

  • FM: σ_min=1e-5, σ_s=0.025, dt=0.05, k_steps=5, α=β=0.001
  • Train: AdamW lr=2e-4, batch=8, 1000 epochs (~60K steps), EMA 0.999
  • Dataset: LOL-v1 our485 train / eval15 eval, 256×256 random crop
  • Eval chop: chop_size=256, chop_stride=224 (train) / 128 (infer)

3.1 experiment_plan_v2_LCFM.md Phase 1 섹션 오류 수정

계획서 Phase 1 설명이 실제 코드와 여러 군데 불일치 → 실제 코드 기준으로 전부 교정. 주요 교정 항목: - Coarse estimator: "3-layer residual CNN (~2M)" → RRDBNet (~5.5M) - Flow ch_mult: [1,2,4,4] → [1,2,1,2] - Flow base ch: 64 → 128 - Flow 입력: "concat(z_t, z_coarse)=32ch" → z_t만 (16ch) - Time conditioning: AdaGN → MLP bias - σ_min: 1e-4 → 1e-5, σ_s: 0.1 → 0.025 - Batch: 16 → 8 - 학습 스케줄: "200K+100K 2-phase" → 1000 epochs 단일 스케줄 - Loss 수식: 단순 velocity-MSE → segment-endpoint consistency + pixel MSE

3.3 ImageSpliterTh — 가로/세로 비대칭 chop 지원

  • 기존엔 chop_size / chop_stride 스칼라만 지원하여 H=W만 가능했음.
  • ELIR/utils.py ImageSpliterTh가 pch_size / stride로 int 또는 (h, w) 튜플을 받도록 수정. 내부에서 H/W 분리, blending mask overlap도 축별로 적용.
  • 호출부(ELIR/irsetup.py, infer.py)에서 config의 chop_size_h / chop_size_w / chop_stride_h / chop_stride_w를 우선 사용, 없으면 기존 chop_size / chop_stride로 fallback (후방 호환).
  • 사용 예: yaml chop: {sf: 1, upscale: 1, chop_size_h: 400, chop_size_w: 600, chop_stride_h: 336, chop_stride_w: 512}
  • LOL-v1이 600×400(H=400, W=600)이므로 Phase 2에서 chop 없이 한 번에 처리하고 싶다면 이 기능을 쓸 수 있음.

4. Phase 2: 다음 세션에서 할 일 (계획 요약)

상위 계획서의 Phase 2 섹션(experiment_plan_v2_LCFM.md L146~) 기반입니다.

4.1 목표

Retinex 분해 후 Illumination(I), Reflectance(R)에 각각 별도의 LCFM을 적용하여 Phase 1 대비 성능 향상을 검증.

4.2 주요 결정 사항 (계획서 권고안)

  • I는 pixel space, R은 latent space에서 처리 (option C)
  • I: 1ch, low-frequency/smooth → latent 이점 적고 AE 3ch 입력과 어색
  • R: 3ch, complex noise/texture → latent flow 유리
  • I-flow: 경량 CNN (~3M), pixel space, 1~2 NFE로 충분 예상
  • R-flow: LUNet (~10M), latent space, 2~4 NFE

4.3 Retinex Decomposer 선택

계획서 권고 순서: 1. Option 2 (max-channel 기반, 학습 불필요) — 1일 내 빠른 검증 2. Option 1 (RetinexNet Decom-Net, pretrained) — 본 실험 3. Option 3 (KinD decomposer) — ablation

4.4 주의사항

  • Decomposer는 반드시 freeze (joint training 시 decomposition degenerate 위험)
  • LOL-v1 485장은 매우 작음 → data augmentation 필수
  • Phase 1과 동일한 TAESD3 AutoEncoder 재사용 가능 (R이 3ch RGB이므로 자연스러움)
  • Phase 1 Encoder Eω / Coarse Estimator는 R-flow용으로 처음부터 다시 학습해야 함 (입력 도메인이 RGB LQ → R_low로 바뀜)

4.5 Ablation (Phase 2 내)

  • Direct LCFM (Phase 1) vs. Retinex + component LCFM
  • I-flow + R에 plain denoiser (IllumFlow 재현) vs. Ours
  • I-flow NFE: 1, 2, 4
  • R-flow NFE: 1, 2, 4, 8
  • Decomposer 3종 비교

4.6 Go/No-Go

  • ✅ Phase 2 > Phase 1 (PSNR ≥ 0.5dB or LPIPS 개선) → Phase 3 (coupled dual-flow)
  • ❌ Phase 2 ≤ Phase 1 → Retinex 포기, Phase 1 + Phase 4(PMRF)로 논문 방향 선회
